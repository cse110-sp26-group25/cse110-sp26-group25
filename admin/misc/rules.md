# Group Contract 

## Code standards: 
- Keep documentation
  - utilize headers 
  - If there’s any subsections / specific functionality portions of your function label them  
- If your code is going of screen please indent 
- Naming convention: **camelCase**  
- If your change doesn’t work please revert it, don’t make some else deal with it 
- **Don’t go rouge**: if you plan to divert from what the team discussed, please let other know / inform the appropriate channels so people aren’t blindsighted

## Git Standards 
- Write **descriptive** git push and commit messages 
- If there are issues with merging please notify anyone who’s branches might be affected  
- Create a new branch for each task 
   - If you’ve been assigned to two tasks, make two separate branches for them  
   - You don’t need to create multiple sub branches for a single task unless you deem it necessary
- Please do not touch repo settings without checking in with the team 
- **Don’t push without making sure your changes work**, and sync correctly with everyone else’s pushes

## Slack Standards  
- Have **notifications** on, at least for @s
  - Will only use mentions for urgent / nessesary messages  
- Expectation to check couple times a day and stay on top of notifications 


## Attendance expectations (Meetings and Lectures)
- Expectation is that you **show up to lecture**   
- If you expect to be absent, **let team leads know beforehand** 
- The onus is on you to check in and figure out what you missed 
   - Ex: ask a teammate about lecture, or check meeting minutes  
- If you are not at the meeting, know that you may have less input on the rolls that you are assigned

## Workload Expectations: 
- If the workload is too much please let team leads / someone know so we can reevaluate and redistribute 
  - People’s workloads differing from week to week is understandable, but this needs to be communicated 
- If the class has a **EOD deadline, treat the deadline as if 8pm** (4 hours earlier)  
  - though the majority of it should be done / pushed the day before - to account for any merge conflicts or similar issues
 
## Social Expectations / Conflict Resolution: 
- Please be **respectful** 
- If there’s anything you're uncomfortable with (ex: colorful language) please let the team or team leads know 
- Don’t take any criticisms as a personal attack, we all mess up some times, and should feel comfortable speaking up if we have any concerns 
- Feel free to contact the team leads, the TA, Powell, etc.. if there are issues you feel cannot be resolved, or don’t feel comfortable addressing personally 
- If asked to keep something confidential it will be kept confidential (bar safely concerns) 
- If work is not getting done expect to be contacted 
- You shouldn’t have to be on call 24/7, however there may be occasions where you are expected to be easily accessible
