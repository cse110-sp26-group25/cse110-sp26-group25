### Notes moving forward
- Bring up task questions/polls during meetings
  - Schedule meetings ahead of time so people can message about stuff they want to talk about
- Be assertive about your portion of the program and what it needs
- Specify task requirements
- Have a small group of people planning and have the rest of the team working
- Create github issues for tasks
- Make title comments for portions of the code because CT.JS ports things over in giant code blocks due to its backend!
  - Document everything inside of the changelog! Include things like:
    1. Code section titles
    2. Stress testing
    3. UI testing
    4. Code by line
- One person working on one module at the same time - if necessary screenshare what is happening or make copies to fiddle with
- Branch things incrementally and make pull requests as soon as possible. We want everyone to be working with the newest versions so that we avoid merge conflicts!
